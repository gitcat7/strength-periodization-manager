# 中文命令映射

## 查询

| 用户表达示例 | 命令 | 说明 |
| --- | --- | --- |
| 今天练什么、查看今日训练、下一次训练 | `今日` | 返回今天或下一次待完成训练及动作记录 ID |
| 查看计划、这周怎么练、我的周期 | `计划` | 返回当前完整周期 |
| 查看训练历史、最近练得怎么样 | `历史 --数量 N` | 默认最近 10 次，最多 50 次 |
| 查看进步、训练量、平均 RPE | `进度` | 汇总最近 20 次已完成训练 |
| 查看 PR、我的冲击目标 | `PR` | 返回当前用户的全部 PR 目标 |

## 写入

### 记录一组

用户需要提供动作、组序号、实际重量、实际次数和 RPE。先运行 `今日`，用动作名称找到准确的 `workout_exercise_id`，再执行：

```bash
python scripts/strength_manager.py 记录组 --动作记录ID UUID --组 1 --重量 75 --次数 5 --RPE 8
```

不要把列表序号当成数据库 ID。重量单位固定为 kg。RPE 必须在 1 到 10 之间。

### 完成训练

从 `今日` 返回值读取 `workout.id`。只有用户明确表示训练已经完成时才执行：

```bash
python scripts/strength_manager.py 完成训练 --训练ID UUID
```

## 常见错误

- `unauthorized`: 令牌缺失、过期或已撤销；让用户在系统设置中重新生成并安全配置。
- `invalid_request`: 缺少字段或数值越界；根据错误补齐参数。
- `not_found`: ID 不属于当前用户或记录不存在；重新读取 `今日`，不要猜测 ID。
- `server_error`: 服务端配置或数据库迁移可能未完成；停止写入并向用户报告错误原文。
