# P0 需求：今日训练与自由训练记录可用性

## 背景

自由训练动作选择目前引用 wger 外部数据。部分结果缺失可读名称时被降级为“其他肌群训练动作 + 编号”等文本，用户无法确认自己将记录什么动作。对新手和初级训练者而言，这会直接阻断训练记录，也损害对计划与数据的信任。

本需求只处理“今日训练/自由训练记录能够安全、清晰地完成”的 P0 阻断项。收藏、模板、动作示范、热身组、离线同步等 P1/P2 项目不在本轮范围。

## 目标

1. 用户只能从可理解、可确认的动作结果中选择动作。
2. 用户能明确知道正在记录自由训练还是周期计划内训练。
3. 用户不能把明显不完整或不合理的训练数据写入完成记录。
4. 用户完成训练前能确认本次记录的核心结果和遗漏。

## 用户与场景

- 新手：从首页选择“快速记录自由训练”，希望快速找到“杠铃卧推”“高位下拉”等常见动作。
- 初级训练者：完成计划训练时逐组填写重量、次数、RPE，希望避免漏填或误结束。
- 自由训练者：临时安排训练，不希望它自动更改下一次周期计划，但希望它保存在历史和进展中。

## P0-1：可读且受审核的动作选择

### 需求

- 建立本地“审核动作库”，仅覆盖新手/初级用户最常用的 80-150 个动作。
- 每个审核动作至少包含：标准中文名、英文别名、器械、主要刺激肌群、次要刺激肌群、动作模式、是否为高技术/高风险动作。
- 默认动作选择页仅展示审核动作库，不展示随机或未审核的 wger 结果。
- wger 仍可作为补充搜索来源，但只有满足以下全部条件的结果才可直接加入训练：
  - 有非空、可读的主名称；
  - 不是由“肌群 + 训练动作 + 编号”组成的兜底名称；
  - 可展示至少一种器械或主要刺激肌群。
- 未审核的外部结果必须明确标记“待确认动作”，不得作为默认列表或搜索首屏结果。
- 任何缺失可读名称的外部结果必须隐藏，不能显示编号兜底名称。
- 搜索排序固定为：标准中文精确名 > 中文常见别名 > 英文名 > 主要刺激肌群 > 其他可读外部结果。
- 搜索结果每项必须展示：标准动作名、器械、主要刺激肌群、动作类型；点击后允许查看简短训练要点或来源信息，再添加到训练。
- 无有效搜索结果或结果质量不足时，提供“手动添加动作”入口；用户至少填写动作名称，器械和训练部位为可选项。手动动作只能进入当前用户自己的训练记录，不写入公共审核动作库。

### 验收

1. 搜索“卧推”时，首屏没有“其他肌群训练动作 + 编号”样式文本。
2. 默认列表中的每一项均有中文名称、器械、主要肌群和类型。
3. 不可读或空名称的 wger 项不能被添加。
4. 无结果时用户可用手动名称创建当前训练中的动作。
5. 同一账号的手动动作不出现在其他账号的搜索结果中。

## P0-2：自由训练与周期计划训练边界

### 需求

- 首页保留并清晰区分两个入口：`继续今日计划` 与 `快速记录自由训练`。
- 自由训练页面顶部固定显示“自由训练”标签和说明：`本次记录会保存到历史与进展，不会自动调整你的周期计划。`。
- 周期计划训练页面顶部显示计划名称、训练方向和计划状态，不使用“自由训练”标签。
- 自由训练完成后，训练历史、进展、CSV 均包含该记录；不改变周期中下一节训练、周期完成状态、计划建议或计划重建结果。

### 验收

1. 用户从首页能在一次点击内选择“继续今日计划”或“快速记录自由训练”。
2. 自由训练页面始终显示边界说明。
3. 完成自由训练后，计划页的下一节训练和周期状态不变。
4. 完成自由训练后，历史、进展和 CSV 可找到该记录。

## P0-3：每组记录字段与保存校验

### 需求

- 每一组必须显示固定字段表头：`重量 (kg)`、`次数`、`RPE`、`完成`。
- 已勾选完成的组，必须填写次数。
- 负重动作的已完成组，必须填写重量和 RPE；自重、辅助重量或用户手动动作可明确选择其负重形式后按对应规则校验。
- RPE 仅允许 1-10 的整数或现有产品支持的合法刻度。
- 重量和次数必须拦截明显异常值；具体边界沿用项目现有安全规则，且不能接受负值、零次数、非数字或超出数据库字段承载范围的值。
- 保存失败或校验失败时，保留用户已输入的重量、次数、RPE、完成状态和备注，并在对应字段附近显示错误。

### 验收

1. 已完成组缺次数时，不能保存或结束训练。
2. 负重完成组缺重量或 RPE 时，不能保存或结束训练。
3. RPE 为 0、11、非数字时被阻止。
4. 校验失败后用户输入不丢失。
5. 合法数据可正常保存，单位固定为 kg。

## P0-4：结束训练前完成摘要

### 需求

- 用户点击“完成训练”时，先显示确认摘要，用户确认后才写入最终完成状态。
- 摘要至少显示：已完成组数、总吨位、最高 e1RM、未完成动作或组数量。
- 若存在未完成组，摘要必须说明具体数量，并提供两个明确动作：`返回继续记录`、`仍然结束训练`。
- 无法计算总吨位或 e1RM 的动作不应阻断完成；摘要显示“该指标不适用”而不是伪造数值。
- 自由训练摘要再次说明不会改动周期计划；计划训练摘要仅展示本次计划训练结果。

### 验收

1. 点击完成训练不会立即关闭或写入完成状态，而是先出现摘要。
2. 摘要中的已完成组数和总吨位与已保存组记录一致。
3. 有漏记时用户能返回继续填写，也能明确选择结束。
4. 高 e1RM 无法计算时页面不报错。

## 数据与权限

- 审核动作库必须是只读的系统数据；浏览器不可直接向第三方动作源请求数据。
- 外部动作、手动动作与其快照继续通过当前用户隔离的训练记录保存，所有查询和写入受现有 Supabase RLS 保护。
- 不同步或持久化完整 wger 动作目录；审核库是产品自有的有限动作定义，不是第三方目录镜像。
- 现有周期计划核心动作与已完成训练历史必须保持可读、可写和可导出。

## 非目标

- 不在本轮实现完整动作视频、图片、动作教学页或外部媒体托管。
- 不在本轮实现自由训练模板、收藏、热身组/顶组/回退组、训练状态自评、实时训练结构诊断、离线同步或训练摘要分享图。
- 不将自由训练自动写入、改写或重排周期计划。

## 实施优先级

1. P0-1 审核动作库与不可读外部动作拦截。
2. P0-3 每组字段和保存校验。
3. P0-4 结束前完成摘要。
4. P0-2 自由训练/周期计划边界与回归验证。

## 总体验收

- 新用户无需计划参数也能从首页开始一场自由训练。
- 用户可以从可读动作中选择、录入合法组数据、查看完成摘要并保存。
- 不存在不可读动作名称、编号兜底名称或无提示的漏记完成。
- 周期计划训练与自由训练互不篡改，但都在用户自己的历史、进展和导出中可见。
- 自动化测试、类型检查、lint、生产构建和认证后浏览器烟测均通过。
