# MRD 市场需求文档

## 1. 产品概述

产品暂名：力训周期管家  
产品形态：移动端优先响应式 Web/PWA  
目标阶段：MVP  
核心人群：健身小白、有一定训练基础但缺少系统规划的人  
核心问题：用户不知道今天练什么、练多重、何时加重量、何时冲 PR、何时降载恢复。

## 2. 市场背景

力量训练正在从“健美式孤立动作”逐渐走向“可量化、可周期化、可追踪”的训练方式。用户会在社交平台看到 5x5、531、线性进阶、RPE、自主调节、PR 等概念，但真正执行时容易遇到断层：

- 知道要记录，但记录工具只是表格或普通笔记。
- 知道要渐进超负荷，但不知道何时加重量。
- 知道要冲 PR，但不知道准备周期怎么安排。
- 知道疲劳会影响表现，但不会把失败、RPE、休息和降载联系起来。
- 下载了健身 App，但很多偏动作库、课程、打卡或内容，不是围绕力量周期来设计。

## 3. 目标用户

### 3.1 新手用户

画像：

- 训练经验 0-12 个月。
- 主要目标是变强、增肌、养成规律训练。
- 常练深蹲、卧推、硬拉、推举、划船、引体向上等基础动作。
- 对 RPE、1RM、周期化概念了解有限。

痛点：

- 不知道该从多轻开始。
- 每次训练临时决定动作和重量。
- 加重量太激进，容易失败或受伤。
- 训练几周后进入平台期，不知道怎么办。

核心需求：

- 今天练什么一眼可见。
- 每组重量、次数和休息时间明确。
- 系统告诉我下次加多少。
- 失败后自动降低难度或安排恢复。

### 3.2 有一定基础的用户

画像：

- 训练经验 1-3 年。
- 已经知道主项、副项、PR、容量、强度等概念。
- 可能用 Excel、Notion、Strong、FitNotes、纸笔记录。

痛点：

- 普通记录工具不会给出周期建议。
- 手动维护表格麻烦，移动端体验差。
- 想安排 PR，但缺乏可执行的准备节奏。
- 不同动作的进阶速度不同，统一模板不够灵活。

核心需求：

- 自定义训练模板。
- 按动作维度追踪 e1RM、训练量、失败率。
- 根据表现调整训练最大值和下周重量。
- PR 尝试前自动安排减量或低疲劳周。

## 4. 用户任务场景

### 4.1 健身房现场记录

用户打开手机，看到今天训练任务：深蹲 5x5 80kg、卧推 5x5 55kg、划船 3x8 50kg。每完成一组快速点选完成，必要时修改实际重量、次数、RPE。训练结束后系统给出“下次建议”。

### 4.2 周计划查看

用户在周日查看下周安排，系统显示 3 天或 4 天训练计划，包括主项、辅助动作、强度、预估训练时长和本周目标。

### 4.3 平台期处理

用户连续两次卧推目标组失败，系统不再机械加重，而是建议下次降低 5%、减少一组或进入恢复周。

### 4.4 PR 安排

用户希望 6 周后测试深蹲 PR，系统将后续训练拆为积累、强化、减量、测试四个阶段，并在测试日给出尝试重量阶梯。

## 5. 竞品与替代方案

### 5.1 普通健身记录 App

优势：

- 记录动作和组数方便。
- 动作库完善。
- 图表成熟。

不足：

- 通常不理解周期计划。
- 自动调重逻辑较弱。
- PR 安排不是核心流程。

### 5.2 表格、Notion、纸笔

优势：

- 灵活、成本低。
- 适合有经验用户自定义。

不足：

- 移动端记录效率低。
- 新手不会设计公式。
- 没有主动提醒和结构化建议。

### 5.3 固定训练模板

优势：

- 上手简单。
- 对新手有明确路径。

不足：

- 对用户表现反馈不敏感。
- 一旦失败、缺勤或临时改动，模板容易失效。

## 6. 机会判断

可切入机会：

- 从“记录工具”升级为“动态计划工具”。
- 先聚焦力量训练主项，不追求全品类健身。
- 用简单规则引擎解决 80% 新手和进阶用户的调重问题。
- MVP 用 PWA 降低开发和运维成本。

核心差异：

- 不是只记录，而是记录后给出下次重量。
- 不是只给模板，而是模板会随表现变化。
- 不是只展示 PR，而是安排 PR 准备路径。

## 7. 市场定位

一句话定位：

一款帮普通训练者自动安排力量训练周期、记录表现并动态调整重量的训练计划工具。

目标切口：

- 先服务以深蹲、卧推、硬拉、推举为核心动作的人。
- 先覆盖 3 天或 4 天训练频率。
- 先支持线性进阶和简化波动周期。

## 8. MVP 成功指标

### 8.1 激活指标

- 新用户完成基础信息配置率 >= 70%。
- 新用户生成第一份计划率 >= 60%。
- 生成计划后完成第一次训练记录率 >= 40%。

### 8.2 留存指标

- D7 留存 >= 25%。
- 4 周内完成 >= 6 次训练的用户占比 >= 20%。
- 用户连续记录 3 周后仍保留计划的占比 >= 30%。

### 8.3 产品价值指标

- 用户训练后接受系统推荐重量的比例 >= 60%。
- 连续失败后触发调整规则的用户中，下一次完成率提升。
- 用户手动修改计划的动作数占比可控，说明默认计划足够可用。

## 9. 市场风险

- 新手可能不理解 RPE、PR、训练最大值，需要降低术语门槛。
- 训练建议涉及安全边界，不能表现得像医疗或专业康复建议。
- 如果调重规则太简单，进阶用户会觉得不够专业。
- 如果配置太复杂，新手会流失。

## 10. MRD 结论

市场上存在大量记录工具和训练内容，但“个人力量周期动态管理”的轻量产品仍有空间。MVP 应聚焦训练计划执行闭环：配置目标、生成计划、记录训练、调整重量、安排 PR。先不要扩展到社交、内容课程和复杂 AI 教练。
