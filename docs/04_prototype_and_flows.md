## 4. 核心用户流程

### 4.1 首次使用流程

```mermaid
flowchart TD
    A[打开产品] --> B[注册/登录]
    B --> C[填写训练画像]
    C --> D[填写主项水平]
    D --> E[选择每周训练天数]
    E --> F[生成4周计划]
    F --> G[查看今日训练]
    G --> H[开始第一次训练]
```

### 4.2 训练记录流程

```mermaid
flowchart TD
    A[进入今日训练] --> B[查看动作目标]
    B --> C[记录每组重量/次数/RPE]
    C --> D{是否完成所有动作}
    D -- 否 --> C
    D -- 是 --> E[点击完成训练]
    E --> F[计算完成率/e1RM/训练量]
    F --> G[生成下次建议]
    G --> H{用户是否采纳}
    H -- 采纳 --> I[更新后续计划]
    H -- 修改 --> J[保存用户修改]
    H -- 拒绝 --> K[保留原计划]
    J --> I
    I --> L[显示训练总结]
    K --> L
```

### 4.3 PR 安排流程

```mermaid
flowchart TD
    A[创建PR目标] --> B[选择动作]
    B --> C[输入目标重量和日期]
    C --> D[读取当前e1RM和训练计划]
    D --> E{距离测试日}
    E -- 大于4周 --> F[安排积累/强化/减量/测试]
    E -- 2到4周 --> G[安排强化/减量/测试]
    E -- 小于2周 --> H[提示周期过短并给保守建议]
    F --> I[写入计划]
    G --> I
    H --> I
```

## 5. 训练完成时序图

```mermaid
sequenceDiagram
    participant U as 用户
    participant UI as 前端
    participant API as 后端API
    participant DB as 数据库
    participant Engine as 规则引擎

    U->>UI: 输入每组训练记录
    UI->>API: 保存训练草稿
    API->>DB: upsert SetLog
    DB-->>API: 保存成功
    API-->>UI: 返回草稿状态
    U->>UI: 点击完成训练
    UI->>API: POST complete workout
    API->>DB: 读取Workout和SetLog
    API->>Engine: 计算完成率/e1RM/推荐重量
    Engine-->>API: 返回Recommendation
    API->>DB: 保存训练完成和推荐
    API-->>UI: 返回总结和下次建议
    UI-->>U: 展示总结页
```

## 7. 设计原则

- 训练现场优先：少字、少层级、大按钮。
- 默认值优先：用户只在异常时修改。
- 建议可解释：每个重量调整都要说明原因。
- 新手少术语：可以显示“强度偏高”，不要一上来堆 RPE/TM/e1RM。
- 进阶不封死：允许用户编辑计划和建议。
