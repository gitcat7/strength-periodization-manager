## 4. 产品决策资料

### 4.1 必须提供

- 产品名称候选 3-5 个，或确认先用临时名。
- 目标用户优先级：小白优先、初级用户优先，还是二者都兼顾。
- MVP 是否只覆盖力量训练，不做减脂、饮食、体态、康复。
- 第一版是否只支持中文。
- 第一版是否只支持 kg。
- 第一版是否需要游客试用。
- 第一版是否需要微信登录。

### 4.2 建议提供

- 你喜欢的竞品或替代工具截图。
- 你不喜欢的健身 App 体验。
- 你希望产品给人的感觉：专业、克制、硬核、简单、陪伴感等。
- 你希望避免的产品风格。

## 8. 每天 22:00 你可以提供什么

你不需要全天在线。每天晚上只需要处理三类事项：

- 决策：确认范围、命名、交互取舍。
- 资源：注册账号、配置平台、提供训练样本。
- 验收：打开预览地址，按我给的步骤测试。

建议固定节奏：

```text
22:00-22:05 看当天成果
22:05-22:15 做关键决策或提供账号资源
22:15-22:25 按测试清单验收
22:25-22:30 确认第二天目标
```
