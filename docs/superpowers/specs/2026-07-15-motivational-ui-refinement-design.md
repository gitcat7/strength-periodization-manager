# 动感训练界面优化设计

## 目标

在不改变训练逻辑、数据模型、导航结构和核心业务文案的前提下，把力训周期管家从“信息正确但视觉平均”的工具界面，升级为更有训练节奏、状态反馈和激励感的移动优先体验。

## 已确认范围

- 视觉气质选择为有运动感、带轻度激励氛围的健身 App。
- 不做等级、徽章、连胜、角色或其他游戏化机制。
- 不添加图片素材、插画、媒体库或大型营销视觉。
- 不增加或删除底部五项导航。
- 不改训练计划算法、动作处方、休息日规则、数据库结构或现有功能入口。
- 必须兼容 320x568、390x844、768x1024、1440x900。
- 必须尊重 prefers-reduced-motion。

## 视觉系统

保留现有绿色 action 作为主要操作和完成态颜色；补充语义色以建立训练场景层级：

- action green：主操作、已完成、可推进状态。
- intensity orange：高强度训练、PR 目标、重要提醒。
- recovery blue-gray：休息/恢复、低负荷说明。
- neutral ink/field：信息和背景层次。
- error red：仅用于错误和不可执行状态。

不使用大面积单色渐变、装饰球、模糊背景、营销式大卡片或无意义插图。卡片圆角保持紧凑，页面区块不嵌套浮层卡片。

## 页面层级

所有主要页面统一为三层：

1. 紧凑标题区：页面名称、当前周期或状态、一个最有价值的辅助指标。
2. 核心行动区：今日训练、生成/重建计划、保存记录、PR 尝试或完成休息。
3. 扫描信息区：趋势、计划条目、历史、建议和设置。

信息区块按重要性使用背景层、细边框、色条或图标区别，避免每个模块都是相同白卡和相同字号。

## 页面改造

### 首页

突出“当前该做什么”：今日训练、今日恢复或下一节训练成为第一视觉对象。摘要指标改为更紧凑的对比排布；训练连续性、近期 PR 和计划状态使用语义色和短标签，不增加额外仪表盘。

### 今日训练

训练标题、动作完成进度和主要动作处方形成一个清晰的执行头部。每组完成、保存记录、结束训练使用短暂、可感知的状态反馈；组间休息计时以进度条和高可读数字为主。动作卡片保持稳定高度，避免填写数据时页面跳动。

### 计划

训练日、休息/恢复日、已完成、下一节和未来日程拥有不同的视觉密度与语义色。计划重建确认抽屉强调影响范围、确认操作和提交中状态；不使用破坏性的醒目警告色来表达普通重建。

### 进展与 PR

进度图和 PR 卡片优先强化变化方向、当前值和下一步，不堆叠装饰元素。强度提升使用暖橙，恢复/降载使用蓝灰，只有异常状态使用红色。

### 历史与设置

历史记录强调日期、完成状态和训练摘要；休息日呈现为低强调恢复条目。设置页按账户、数据、外观/体验等主题分组，避免长串同级按钮。

## 交互与动效

- 按钮提供 120-180ms 的按压和禁用反馈。
- 页面初始加载使用稳定的骨架占位，减少白屏和布局跳动。
- 成功操作显示一次简短的颜色/图标确认，不弹出阻塞式庆祝层。
- 计划筛选、底部导航和分段选择使用平滑色彩/位置过渡。
- reduced motion 下取消位移、缩放和循环动画，只保留状态颜色变化。
- 键盘焦点始终清晰可见；对话框和抽屉保持焦点恢复行为。

## 验收

1. 核心页面在四个目标视口没有横向滚动、文字遮挡、控件重叠或跳动。
2. 首页、今日、计划、进展、PR、历史、设置的行动层级在首屏可识别。
3. 训练、恢复、高强度、完成和错误状态颜色一致且可区分。
4. 所有现有业务流程、训练记录和导航链接保持可用。
5. reduced motion 下没有非必要位移或循环动画。
6. 验收使用浏览器截图和控制台检查，并运行测试、类型检查和生产构建。

## 非目标

- 不改品牌名称或 Logo。
- 不做深色模式。
- 不增加社交、排行榜、徽章、成就或打卡连胜。
- 不改变数据导出格式、训练周期生成逻辑或 Fitness Coach 建议。
