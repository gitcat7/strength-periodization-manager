# CC 交付摘要模板

## 状态

只能写“完成”“部分完成”或“阻塞”，并说明依据。

## Git

- 分支：当前 `claude/*` 分支
- 提交：完整提交哈希
- 工作区：是否干净；若不干净，列出文件

## 变更

逐项列出修改文件和每个文件的行为变化，不只写文件名。

## 验证

逐条列出实际运行的命令、退出码、通过/失败数量和关键输出。没有运行的命令必须明确写“未运行”。

## 关键决策

说明实现如何满足委派文件中的接口、安全和产品约束。

## 风险与未完成项

列出已知风险、未覆盖环境、人工验证需求和任何超出 CC 权限的后续步骤。没有也要明确写“无已知项”。
