# CC 任务委派模板

复制本文件创建新的 `docs/agent-collaboration/tasks/YYYY-MM-DD-task-name.md`，发送前必须把每节改成该任务的确定内容。

## 背景与目标

说明用户问题、目标行为和它为什么值得做。目标必须能被一个独立测试验证。

## 基线与分支

写明基线分支、CC 分支、Worktree 绝对路径，以及开始前预期的 `git status --short --branch` 输出。

## 允许范围

逐项列出允许创建或修改的文件，以及每个文件的职责。

## 禁止事项

列出不能修改的模块、不能扩大的产品范围，以及推送、部署、生产数据库和密钥边界。

## 行为与接口

写明输入、输出、函数或类型签名、UI 文案、数据库/API 参数和失败行为。不能使用“适当处理”等含糊描述。

## TDD 与验收标准

列出先失败后通过的具体测试，以及每一条用户可观察的验收结果。

## 验证命令

给出可直接复制的完整命令和预期退出结果。

## Git 与交付

给出唯一提交信息，明确禁止推送，并要求按 `cc-handoff-template.md` 报告。
