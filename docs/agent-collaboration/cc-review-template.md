# CC 只读审查模板

## 审查目标

审查指定提交相对于指定基线的差异，只报告能被代码、测试或项目文档证明的问题。

## 规则

- 只读，不修改文件、不提交、不推送。
- 先检查需求和验收标准，再检查 diff。
- 每条发现包含严重级别 P0–P3、文件、最小行号范围、复现条件、实际影响和建议方向。
- 优先报告正确性、安全、用户隔离、数据损坏、回归和缺失测试。
- 不报告纯个人风格偏好。
- 如果没有可操作问题，明确写“未发现可操作问题”，并列出仍未验证的风险。

## 输出顺序

1. Findings：按 P0 到 P3 排序。
2. Open questions：仅列确实阻止判断的问题。
3. Verification gaps：列未运行或无法运行的验证。
4. Summary：一段话说明提交是否适合进入 Codex 集成审查。
